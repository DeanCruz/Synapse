# 2.3 Projects and Knowledge Graph

Synapse is standalone. It can manage any target project without living inside that project.

## Project Roots

Each dashboard has a project root. This is the directory workers read and modify.

Set it from the app:

1. Select a dashboard.
2. Click the Project button in the dashboard action bar.
3. Choose your project directory.

When you create a Code dashboard from the sidebar `+`, Synapse opens a folder picker first, then creates a dashboard already bound to that folder.

You can also use project commands when working from chat:

```text
!project
!project set /path/to/your/project
```

## Additional Context Directories

Additional context directories are read-only. Use them for files that help agents understand the work but should not be edited:

- Shared packages.
- API documentation.
- Design files exported as docs.
- A related backend or frontend repo.

Workers receive these paths as reference material. They should write code only in the selected project root.

## Project Instructions

A good project instruction file tells agents:

- Tech stack.
- File structure.
- Commands for build, test, lint, and dev server.
- Coding conventions.
- Safety rules.
- Patterns to reuse.

Run this to create or refresh one:

```text
!scaffold
```

For full setup, run:

```text
!initialize
```

## Project Knowledge Graph

The Project Knowledge Graph lives in:

```text
{project_root}/.synapse/knowledge/
```

It stores durable project understanding:

- File purposes.
- Domains and tags.
- Relationships between files.
- Gotchas.
- Conventions.
- Patterns.
- Swarm-level lessons.

Agents use this graph to find relevant code faster and avoid rediscovering the same project details every swarm.

## Build The Knowledge Graph

Run:

```text
!learn
```

This dispatches a swarm that reads significant project files and writes structured knowledge into `.synapse/knowledge/`.

For large projects, run `!learn` before the first major implementation swarm. Planning takes longer once, but future work gets better context.

## Refresh The Knowledge Graph

Run:

```text
!learn_update
```

Use this after:

- A large refactor.
- New files were added.
- Many files changed since the last swarm.
- Agents seem to be relying on stale project assumptions.

`!learn_update` focuses on stale or new files. If too much changed, it may recommend a full `!learn`.

## Query Project Context

Use `!context` before a risky change:

```text
!context "payment webhook validation"
!context "where user permissions are enforced"
```

Synapse checks the Project Knowledge Graph first, then supplements with file search when needed.

## Gitignore The Synapse Metadata

For most projects, add this to the project's `.gitignore`:

```text
.synapse/
```

The knowledge graph is local operational metadata. It usually should not be committed with application code.
