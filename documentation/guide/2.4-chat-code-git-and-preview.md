# 2.4 Chat, Code, Git, and Preview

Synapse combines chat, code inspection, git workflows, and app preview so you can keep one loop while agents work.

## Chat

Use chat for:

- Asking how a project works.
- Running `!commands`.
- Starting swarms.
- Reviewing final summaries.
- Continuing a conversation about a dashboard or project.
- Opening Guide or Commands from the header when you need help.

Chat conversations persist across sessions. Multiple chat tabs can belong to the same project, which is useful when you want separate investigation threads.

## Code Explorer

Use Code Explorer when you want to inspect or edit files directly.

Common flow:

1. Open the Code view.
2. Select or create a dashboard bound to your project.
3. Open Code Explorer.
4. Browse the selected dashboard project's file tree.
5. Open files in tabs.
6. Edit and save with the Monaco editor.
7. Use the bottom panel for terminal, output, problems, debug console, and ports.

Code Explorer follows the selected dashboard's project path. It supports lazy-loaded folders, syntax diagnostics, dirty-file indicators, project search, editor tabs, and a Node.js debug workflow.

## Git Manager

Use Git Manager to review and commit changes after agents finish.

Common flow:

1. Open the Git view.
2. Select a dashboard bound to the project you want to review.
3. Choose one of the discovered root or nested git repositories.
4. Review changed files.
5. Click a file to inspect the diff.
6. Stage the files you want.
7. Write a commit message.
8. Commit, then push when ready.

Git Manager discovers git repositories under the selected dashboard project. It also supports branch switching, branch creation, merge previews, fetch, pull, push, stash, and quick actions.

Be careful with destructive actions such as discarding changes or deleting branches. Synapse shows safety dialogs, but you should still review the working tree first.

## Live Preview

Live Preview embeds your running web app inside Synapse.

Setup:

1. Run instrumentation on your project:

```text
!instrument
```

2. Start your app's dev server, such as:

```bash
npm run dev
```

3. Open the Preview tab.
4. Enter the dev server URL or use detected dev-server information when available.
5. Double-click labeled text to edit it inline.

The inline edit workflow depends on `data-synapse-label` attributes. `!instrument` adds those labels to supported JSX, TSX, and HTML text elements.

## How These Views Fit Together

Use the dashboard to monitor agent work. Use Code Explorer to inspect the resulting files in the selected dashboard project. Use Git Manager to review and commit. Use Preview to validate visible app changes.

For user-facing UI work, a good loop is:

1. Start a tracked swarm with clear UI requirements.
2. Watch the dashboard for deviations or failures.
3. Open Preview to check the running app.
4. Use Code Explorer for inspection or small follow-up edits.
5. Use Git Manager to review the diff and commit.
