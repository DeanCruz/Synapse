# 1.0 Intro

Synapse is a desktop control system for working with AI agents across real software projects. You use it to chat with an agent, plan parallel work, watch workers complete tasks, inspect code, manage git changes, preview web apps, and read this guide from one place.

The core idea is simple:

1. Pick or create a dashboard for the project you want Synapse to work on.
2. Bind that dashboard to the project folder.
3. Describe the work in chat.
4. Use serial commands for small questions or `!p_track` for multi-part work.
5. Review the plan.
6. Watch workers report progress on the dashboard.
7. Inspect the final code, git diff, and summary before you commit.

Synapse keeps two locations separate:

| Path | What it means |
|---|---|
| `{tracker_root}` | The Synapse app repository and dashboard data. |
| `{project_root}` | The project agents read from and modify. |

In normal use, you do not need to manage those paths manually. You select a project in the app, and Synapse passes the correct paths to the agents it dispatches.

## What You Can Do

- Run a normal agent chat for focused questions or small edits.
- Start a tracked swarm with `!p_track` when a task has several independent parts.
- Use the dashboard to monitor stages, dependencies, milestones, deviations, and failures.
- Use the Commands browser to discover available `!commands`.
- Use the Guide button to read these markdown docs inside the app.
- Use Code Explorer to browse and edit the selected dashboard's project.
- Use Git Manager to review, stage, commit, pull, and push changes for the selected dashboard's project.
- Use Live Preview to inspect a running web app from the selected project and edit labeled text inline.
- Build a Project Knowledge Graph in `.synapse/knowledge/` so future agents understand your project faster.

## When To Use Synapse

Use Synapse when a task benefits from planning, visibility, or parallel execution:

- Adding a feature that touches frontend, backend, tests, and docs.
- Refactoring across several modules.
- Running code review or project health analysis.
- Asking agents to investigate a large unfamiliar codebase.
- Coordinating multiple independent implementation tasks.

For one-file fixes, a regular chat request is usually enough. For work with three or more independent pieces, use `!p_track`.
