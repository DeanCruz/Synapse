# 2.0 Getting Started

This guide assumes Synapse is already open. It focuses on the first things to do inside the app: choose where to work, understand Chat and Code, initialize a project, and start using agents.

## Pick Your Starting Page

Synapse opens in Chat mode. Use the mode menu in the app header to move between Chat and Code. The header also exposes Archive, Guide, Commands, and the active-agent count.

| Page | Use It For |
|---|---|
| Chat | Talking to agents, asking questions, running `!` commands, starting swarms, and reading final summaries. |
| Code | Watching dashboards, editing files, reviewing git changes, and previewing a running app. |

Start in Chat when you want Synapse to reason about the project or do work for you. Start in Code when you want to browse, edit, run, debug, or review the project yourself.

## Chat Page

The Chat page is the command center for agents. Use it to:

- Ask how the project works.
- Run setup commands such as `!initialize` and `!learn`.
- Start tracked swarms with `!p_track`.
- Continue separate investigation threads in different chat tabs.
- Review worker summaries, failures, and follow-up recommendations.

Each chat is bound to dashboard data so agent activity has a visible place to report progress. For normal project work, you can stay in Chat until you need to inspect files or review changes.

## Code Page

The Code page is the hands-on project area. It contains:

- Dashboards for watching swarm progress, logs, dependencies, and worker state.
- Code Explorer for browsing and editing the selected dashboard's project.
- Git Manager for reviewing diffs, staging, committing, branching, pulling, and pushing discovered git repositories in that project.
- Live Preview for opening a running web app from that project and editing labeled text inline.

Use Code after an agent finishes work, when you want to verify what changed, run commands in the terminal, inspect files, or commit the result.

## Create Or Open A Dashboard

1. Open Synapse.
2. Use the Code sidebar to create or select a dashboard.
3. To create a new Code dashboard, click the sidebar `+` and choose a folder in the picker. Synapse creates the dashboard and binds it to that project.
4. To change an existing dashboard, select it and click the Project button in the dashboard action bar.
5. Select the project directory you want agents to work on.

Each dashboard remembers its own project. You can keep separate dashboards for separate repositories.

## Select The Project

Click the Project button in the dashboard action bar and choose the folder agents should work in. This selected folder becomes the project root for chat commands, Code Explorer, dashboards, Git Manager, Preview, and worker agents.

You can also add optional context directories in the project selector. These are read-only reference folders for agents, such as:

- Shared libraries.
- Design specs.
- Documentation repositories.
- Related projects.

Agents can read additional context, but code changes should happen only in the selected project root.

## Initialize The Project

For a new or newly connected project, run this in Chat:

```text
!initialize
```

`!initialize` prepares the project for Synapse. It detects the tech stack, creates the project `.synapse/` directory, records project metadata, and can create project agent instructions if they are missing.

Run this once per project. It is designed to be safe to rerun; existing project setup is detected and skipped where possible.

## Build Project Knowledge

After initialization, run:

```text
!learn
```

`!learn` builds the Project Knowledge Index, which is Synapse's map of your codebase. It helps future agents find the right files, understand project patterns, avoid known gotchas, and write better worker prompts.

Use `!learn` when setting up a project for the first time or when the project has changed heavily. For smaller follow-up changes, use:

```text
!learn_update
```

`!learn_update` refreshes only stale or new knowledge entries, so it is faster than rebuilding the full index.

## Run Your First Tracked Swarm

Use `!p_track` when the work has multiple independent pieces:

```text
!p_track Add a user settings page with profile editing, password change, notification preferences, and tests. Follow the existing component and API patterns.
```

Synapse will plan the work, populate the dashboard, ask for approval, and then dispatch workers.

## Ask A Small Question Or Request

For a small question or single-file change, type a normal chat request instead of starting a swarm:

```text
Explain how authentication works in this project.
```

Use the smallest mode that fits the work. Swarms are powerful, but they add planning overhead.

## First-Use Flow

For most projects, the first session looks like this:

1. Open Chat.
2. Create or select a dashboard, then bind its project with the sidebar folder picker or the dashboard action bar Project button.
3. Run `!initialize`.
4. Run `!learn`.
5. Ask a question or start a swarm with `!p_track`.
6. Switch to Code to inspect files, review the dashboard, preview the app, or commit changes.
