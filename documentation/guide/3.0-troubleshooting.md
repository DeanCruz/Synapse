# 3.0 Troubleshooting

Use this guide when Synapse does not behave the way you expect.

## The App Does Not Launch

Check requirements:

```bash
node --version
npm --version
```

Then reinstall and start:

```bash
npm install
npm start
```

If native modules fail during install, install Xcode Command Line Tools:

```bash
xcode-select --install
```

## Agents Are Working In The Wrong Project

Check the dashboard's project:

1. Select the dashboard.
2. Click the Project button in the dashboard action bar.
3. Confirm the directory.

If you are creating a new Code dashboard, use the sidebar `+` and choose the correct folder in the picker so the dashboard is bound to the right project from the start.

You can also run:

```text
!project
```

If needed, set the project explicitly:

```text
!project set /path/to/your/project
```

## The Plan Looks Wrong

Do not approve a bad plan. Clarify the prompt and rerun it.

Improve the prompt with:

- Target files or modules.
- Required behavior.
- What should not change.
- Testing expectations.
- Known constraints.

For a focused pre-check, run:

```text
!context "the area I want changed"
!plan "the change I am considering"
```

## A Worker Is Stuck

First inspect the dashboard card. Check its current stage, latest milestone, logs, and dependencies.

Then run:

```text
!inspect {task_id}
!logs --task {task_id}
```

If the task is blocked by another task, inspect the upstream task. If the worker failed, use `!retry {task_id}` when the failure is isolated.

## A Task Failed

Run:

```text
!inspect {task_id}
```

Look for:

- The failure message.
- Files changed before failure.
- Deviations.
- Whether downstream tasks are blocked.

Retry only when the failure is understood:

```text
!retry {task_id}
```

If multiple related tasks failed, ask Synapse to replan with the failure context instead of retrying each task blindly.

## The Dashboard Shows Stale Data

Try:

```text
!status
!logs --last 20
!update_dashboard
```

If the app still appears stale, restart Synapse:

```bash
npm start
```

For active swarms, avoid deleting dashboard data unless you are intentionally canceling or resetting work.

## Agents Miss Important Project Patterns

Build or refresh the Project Knowledge Graph:

```text
!learn
!learn_update
```

Also check that the project has useful instructions:

```text
!scaffold
```

Then rerun the task with a more specific prompt.

## Live Preview Does Not Edit Text

Check:

1. Your app dev server is running.
2. The Preview URL is correct.
3. The page contains supported text elements.
4. `!instrument` has been run.
5. The files are inside the selected project root.

Run:

```text
!instrument
```

Then restart your dev server and reload Preview.

## Git Manager Does Not Show Expected Changes

Check:

- The selected dashboard is bound to the correct project.
- The repository has a `.git` directory.
- Git Manager has selected the expected discovered root or nested repository.
- Changes were saved in Code Explorer or your external editor.
- The file is not ignored by `.gitignore`.

Use fetch or refresh if remote state looks stale. Review diffs before staging or discarding anything.
