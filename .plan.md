# Plan: Add AskUserQuestion Support to Synapse Chat

## Problem
When Claude Code is spawned via Synapse's chat (ClaudeView), it runs with `--print --output-format stream-json`. The `--print` flag makes it non-interactive, so `AskUserQuestion` tool calls fail with an error. The questions never reach the user.

## Solution
Intercept `AskUserQuestion` tool_use events in the UI stream, render them as interactive question cards, and send the user's answers back as a follow-up message via the existing `--resume` mechanism.

**No changes needed to**: ClaudeCodeService, ipc-handlers, or preload. The existing follow-up message flow handles everything.

## Files to Modify

### 1. `src/ui/components/ClaudeView.jsx`

**a) Add `AskUserQuestionBlock` component** (alongside existing `ToolCallBlock`, `ThinkingBubble`, etc.)
- Renders each question with header, question text, and clickable option buttons
- Supports `multiSelect` (toggle multiple) vs single-select
- Shows option labels and descriptions
- Submit button sends answers
- After submission: disables options, shows selected state
- Calls a `sendText()` function with formatted answers

**b) Modify `ConversationMessage`** to accept an `onSendAnswer` prop
- When `msg.type === 'tool_call'` and `msg.block.name === 'AskUserQuestion'`, render `AskUserQuestionBlock` instead of `ToolCallBlock`
- Pass `onSendAnswer` callback through

**c) Pass `sendText` reference** from the main ClaudeView render to `ConversationMessage` via props

### 2. `src/ui/styles/index.css`

Add CSS for the question block:
- `.claude-ask-question` — card container matching tool_call style
- `.claude-ask-header` — question header with tag chip
- `.claude-ask-option` — clickable option button with hover/selected states
- `.claude-ask-submit` — submit button
- `.claude-ask-answered` — disabled state after submission

## Answer Format

When the user submits, send a follow-up message like:
```
Here are my answers to your questions:

**Platform**: macOS only
**Updates**: Auto-update (Recommended)
```

Claude will receive this as a new turn via `--resume` and continue with the answers.

## Flow
```
Claude outputs tool_use "AskUserQuestion" →
  CLI fails (--print mode) → error tool_result →
  UI detects AskUserQuestion block in processEvent →
  Renders interactive question card →
  User clicks options → clicks Submit →
  sendText() fires with formatted answers →
  New --resume process receives answers →
  Claude continues
```
